# 痛点映射标准（必选项）

每次生成解决方案后，必须将功能模块与客户原始痛点逐一对应，形成痛点映射表。

---

## 映射流程

1. **提取痛点** → 从客户原始需求中识别核心痛点（显性 + 隐性）
2. **匹配功能** → 将每个痛点匹配到功能拆分表中的对应功能
3. **标注状态** → 判断覆盖程度：✅ 完全覆盖 / ⚠️ 部分覆盖 / ❌ 未覆盖
4. **生成结论** → 对未覆盖项给出原因说明或补充建议

---

## 痛点识别维度

| 维度 | 常见痛点关键词 | 说明 |
|------|-------------|------|
| 效率 | 人工操作、耗时长、重复劳动、流程繁琐 | 工作效率瓶颈 |
| 数据 | 数据孤岛、信息不互通、数据丢失、报表不准 | 数据管理问题 |
| 协同 | 跨部门难协调、沟通成本高、响应慢 | 部门协作问题 |
| 安全 | 数据泄露、权限失控、审计缺失 | 安全合规问题 |
| 成本 | 维护成本高、投入大、浪费资源 | 成本控制问题 |
| 扩展 | 难以扩展、无法支撑业务增长、技术落后 | 技术架构问题 |
| 体验 | 操作复杂、界面不友好、学习成本高 | 用户体验问题 |

## 隐性问题识别

| 隐性痛点 | 识别依据 | 对应功能 | 覆盖状态 |
|---------|---------|---------|---------|
